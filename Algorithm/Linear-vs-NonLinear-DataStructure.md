## 개념 정의
선형 자료구조와 비선형 자료구조의 차이는 데이터 간 연결 관계에 있다. 선형 자료구조는 각 원소가 앞뒤 1:1 관계를 가지는 구조이다. 비선형 자료구조는 한 노드가 여러 노드와 연결되는 1:N 또는 N:N 관계를 가지는 구조이다. 선형은 순차 접근이 중심이고, 비선형은 계층 또는 네트워크 탐색이 중심이다.

## 동작 원리
1. 선형 자료구조는 시작 지점에서 다음 원소로 이동하며 순서대로 처리한다.
2. 비선형 자료구조는 현재 노드에서 여러 갈래로 분기하며 탐색 경로를 선택한다.
3. 그래서 선형은 인덱스/포인터 이동이 핵심이고, 비선형은 DFS/BFS 같은 탐색 전략이 핵심이다.

처음에는 둘 다 "저장 방식 차이"로 보였는데, 실제로는 "탐색 방식 차이"가 문제 풀이에서 더 큰 기준이라는 점이 보인다.

## 코드
```js
// 선형: 배열 순차 탐색
const arr = [10, 20, 30, 40];
for (const v of arr) {
  console.log(v);
}

// 비선형: 트리 DFS 탐색
const tree = {
  value: "A",
  children: [
    { value: "B", children: [{ value: "D", children: [] }] },
    { value: "C", children: [] },
  ],
};

function dfs(node) {
  console.log(node.value);
  for (const child of node.children) dfs(child);
}

dfs(tree); // A -> B -> D -> C
```

`for...of` 순회는 배열의 인접 원소를 앞에서 뒤로 한 줄로 처리한다. `dfs`는 현재 노드 기준으로 하위 노드를 재귀 호출하며 계층을 따라 내려간다. 전자는 순차 흐름이 고정되어 있고, 후자는 분기 수에 따라 방문 순서가 달라진다. 같은 "순회"라도 실행 모델이 다르다는 점이 핵심이다.

## 언제 쓰는지
- 배열, 연결 리스트, 스택, 큐처럼 순서 기반 처리와 단순 구현이 필요한 경우에 선형 자료구조를 쓴다.
- 조직도, 폴더 구조, 추천 그래프, 경로 탐색처럼 분기/연결 관계를 모델링할 때 비선형 자료구조를 쓴다.
- 단순 순차 문제인데 트리/그래프 모델을 억지로 적용하면 구현 복잡도만 올라가므로 피하는 편이 낫다.

## 핵심 한 줄
선형 자료구조는 "한 줄 순서 처리"에 강하고, 비선형 자료구조는 "분기 관계 탐색"에 강한 구조이다.
