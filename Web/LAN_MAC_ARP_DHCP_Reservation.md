# LAN과 MAC 핵심 정리

---

## 1) LAN은 무엇인가?

### 정의
`LAN(Local Area Network)`은 집, 학교, 회사처럼 가까운 범위에서 기기들이 연결된 내부 네트워크다.

예시 구조:

```text
인터넷 (WAN)
   ↓
공유기
   ↓
[ LAN 영역 ]
   ├── 데스크탑
   ├── 맥북
   ├── 휴대폰
   └── 태블릿
```

특징:
- 사설 IP 사용 (예: `172.30.1.x`)
- 외부에서 직접 접근 불가(기본 설정)
- 공유기가 내부 네트워크를 관리

반대 개념:
- `WAN(Wide Area Network)` = 인터넷 같은 광역 네트워크

---

## 2) MAC은 무엇인가?

### 정의
`MAC(Media Access Control address)`은 네트워크 인터페이스(랜카드)에 부여되는 물리 주소다.

형식 예:

```text
10:E6:6B:02:D1:E0
```

비유:
- IP = 네트워크에서 쓰는 논리 주소(집 주소)
- MAC = 장비 식별용 물리 주소(장비 고유 번호)

참고:
- 일반적으로 MAC은 장비 단위로 고정되어 보이지만, 환경에 따라 변경(스푸핑)도 가능하다.

---

## 3) LAN에서 IP와 MAC은 어떻게 같이 쓰이나?

LAN 통신에서는 실제 전송 대상 식별을 위해 `IP -> MAC` 변환 과정이 필요하다.

이 변환 프로토콜이 `ARP(Address Resolution Protocol)`이다.

예시:

```text
데스크탑
IP:  172.30.1.45
MAC: AA-BB-CC-DD-EE-FF

맥북
IP:  172.30.1.78
MAC: 11-22-33-44-55-66
```

- 공유기/기기는 IP 기준으로 상대를 찾고
- 실제 프레임 전달은 MAC 기준으로 처리한다

---

## 4) 포트포워딩과 MAC이 연결되는 이유

포트포워딩은 내부 목적지 IP가 바뀌면 규칙이 깨질 수 있다.

그래서 공유기에서 `DHCP 예약`을 자주 사용한다.

의미:

```text
"이 MAC 주소에는 항상 같은 내부 IP를 할당"
```

예:

```text
AA-BB-CC-DD-EE-FF -> 172.30.1.45 고정
```

이렇게 하면 재부팅 후에도 서버 PC의 내부 IP가 유지되어 포트포워딩 규칙이 안정적으로 동작한다.

---

## 한 줄 요약

- `LAN` = 집 내부 네트워크
- `MAC` = 장비 물리 주소
- `IP` = 논리 주소
- `ARP` = IP를 MAC으로 변환
- `DHCP 예약` = MAC 기준으로 내부 IP를 고정
