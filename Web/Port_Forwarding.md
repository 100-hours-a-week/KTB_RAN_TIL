# Port Forwarding (포트포워딩)

---

## 1. 기본 네트워크 구조

가정용 공유기 환경은 보통 다음 구조다.

```text
[인터넷]
    ↓
[공유기]   ← 공인 IP (예: 112.186.28.202)
    ↓
[내 PC]    ← 사설 IP (예: 172.30.1.45)
```

- 공인 IP: 인터넷에서 보이는 주소
- 사설 IP: 내부 네트워크(집/회사)에서만 쓰는 주소

---

## 2. 기본 설정에서 막히는 이유

외부 사용자가 `112.186.28.202:25565`로 접속하면, 공유기는 이 요청을 내부 어떤 기기로 보내야 할지 알 수 없다.

- 내부에는 PC, 스마트폰, 태블릿, TV, 콘솔 등 여러 장비가 동시에 연결됨
- 목적 내부 기기를 지정하지 않으면 오동작 위험이 있음
- 그래서 기본값은 `외부 -> 내부 직접 연결 차단`

---

## 3. 포트포워딩의 정의

포트포워딩은 공유기에 다음 규칙을 명시하는 설정이다.

```text
"특정 포트로 들어온 요청을
내부 특정 IP로 전달"
```

예:

```text
25565 포트 요청 -> 172.30.1.45:25565로 전달
```

즉, `인터넷 -> 공유기 -> 내 PC` 경로를 지정해주는 기능이다.

---

## 4. 포트(Port)란

IP가 "집 주소"라면, 포트는 "호수"에 해당한다.

| 서비스 | 포트 |
| --- | --- |
| HTTP/웹 | 80 |
| HTTPS/웹 | 443 |
| Minecraft | 25565 |
| FTP | 21 |

IP는 기기를 식별하고, 포트는 그 기기 안의 어떤 프로그램(서비스)인지 식별한다.

---

## 5. 왜 기본적으로 닫혀 있는가 (보안)

모든 포트가 자동 개방되어 있으면 다음 위험이 커진다.

- 무차별 접속 시도(브루트포스)
- 악성코드/랜섬웨어 유입 가능성 증가
- DDoS 공격 표면 확대

그래서 공유기는 기본적으로 외부 진입을 차단하고, 필요한 포트만 직접 열어야 한다.

---

## 6. 포트포워딩을 열었을 때

장점:
- 외부 사용자 접속 가능
- 게임 서버/개인 서버 운영 가능

단점:
- 개방한 포트가 외부에 노출됨
- 서비스 보안 설정이 필수

실무적 최소 수칙:
- 필요한 포트만 열기
- 화이트리스트 사용
- 온라인 모드 유지(게임 서버)
- 관리자 권한(OP) 최소화
- 서버/플러그인 최신 업데이트 유지

---

## 한 줄 요약

포트포워딩은 공유기에
`"특정 포트 요청을 내부 특정 PC로 전달하라"`
고 알려주는 설정이다.

---

## 연결 개념

- NAT: 사설 IP와 공인 IP 사이 주소 변환
- Port Forwarding: NAT 환경에서 외부 유입을 허용하기 위한 예외 규칙
