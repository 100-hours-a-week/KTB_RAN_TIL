# 집 네트워크 핵심 구조: DHCP, NAT, SNAT, DNAT

---

## 1) 집 안 기기 IP는 서로 같은가?

같은 대역을 쓰더라도 기기별 내부 IP는 다르다.

```text
공유기:   172.30.1.254
데스크탑: 172.30.1.45
맥북:     172.30.1.78
폰:       172.30.1.101
```

- 위 주소들은 `사설 IP(Private IP)`
- 집/회사 내부에서만 사용

---

## 2) 내부 IP는 누가 정하나?

보통 공유기가 DHCP 서버 역할을 하며 자동 배정한다.

### DHCP
- 의미: `Dynamic Host Configuration Protocol`
- 역할: 네트워크에 접속한 기기에 IP를 자동 할당

동작 예:

```text
1. 기기: "IP 주소 주세요"
2. 공유기(DHCP): "172.30.1.45 사용하세요"
3. 기기: "할당 완료"
```

IP가 겹치지 않는 이유:
- 공유기가 현재 사용 중인 주소를 관리
- 이미 할당된 주소는 재할당하지 않음

---

## 3) 내부 -> 공인으로 바꾸는 것: NAT

내부 기기는 사설 IP를 쓰기 때문에, 인터넷 통신 시 공유기가 공인 IP로 변환한다.

예:

```text
172.30.1.45 -> 112.186.28.202
```

여러 내부 기기가 하나의 공인 IP를 공유할 수 있게 해주는 핵심 기술이 NAT다.

---

## 4) 공인 -> 내부로 보내는 것: DNAT(포트포워딩)

기본 NAT는 내부에서 먼저 시작한 연결의 응답만 돌려준다.

- 내부 시작 연결: 허용
- 외부 시작 연결: 기본 차단

외부에서 먼저 접속해야 하는 서버(게임 서버 등)는 `DNAT/포트포워딩` 규칙이 필요하다.

예:

```text
112.186.28.202:25565
        ↓ DNAT (Port Forwarding)
172.30.1.45:25565
```

---

## 5) 용어 구분

| 개념 | 설명 |
| --- | --- |
| NAT | IP 변환 전체 개념 |
| SNAT | 내부 -> 외부 주소 변환 |
| DNAT | 외부 -> 내부 목적지 변환 |
| 포트포워딩 | DNAT의 대표적인 형태 |

---

## 6) 전체 구조 한 번에 보기

```text
[인터넷]
    ↓
공인 IP (112.186.28.202)
    ↓
공유기 (NAT 장치)
    ↓
내부 기기
 ├─ 172.30.1.45 (데스크탑)
 ├─ 172.30.1.78 (맥북)
 └─ 172.30.1.101 (폰)
```

- 내부 IP 배정: DHCP
- 내부 -> 외부: SNAT
- 외부 -> 내부 유입 허용: DNAT(포트포워딩)

---

## 핵심 한 줄 정리

- 내부 IP는 공유기의 DHCP가 자동 배정
- 내부 -> 공인은 SNAT
- 공인 -> 내부는 DNAT(포트포워딩)
