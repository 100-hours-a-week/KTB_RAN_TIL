# Cookie vs  Storage vs  Session vs Cache


### **Cache (캐시)**
    
  로컬 PC에 저장되며, **이미 가져온 소스를 다시 로드하지 않기 위해** 사용된다.
  → 주로 이미지, CSS, JS 등의 정적 리소스를 저장해 **네트워크 부하를 줄임.**

---

### **Session (세션)**

  **서버 측에서 사용자 상태를 저장**하여 보안성을 높인다.
  클라이언트는 실제 데이터가 아닌, 서버에 저장된 **세션 ID(Session ID)** 만 보관한다.
  → 서버가 세션 ID로 사용자를 식별하며, **로그인 유지** 등에 활용된다.

---

### **Cookie (쿠키)**

  클라이언트와 서버 간의 **상태 정보를 주고받기 위한 데이터 조각.**
  브라우저가 요청을 보낼 때마다 **HTTP 헤더에 쿠키를 포함**해 전송한다.
  서버는 이를 통해 “이 요청이 누구의 것인지”를 식별할 수 있다.

  > **퍼스트파티 쿠키 (1st Party Cookie)**

* 생성된 쿠키의 **도메인과 경로가 현재 접속한 사이트와 동일**하다.
    * 예: `myapp.com`에서 생성된 쿠키를 `myapp.com`이 직접 사용


  > **서드파티 쿠키 (3rd Party Cookie)**
    
* **도메인이나 경로가 다른 외부 서비스**에서 생성된 쿠키.
        * 광고 추적, 소셜 로그인 등에 사용
    * 예: `myapp.com` 페이지에서 `facebook.com`의 추적 쿠키가 동작

---

### **Storage (스토리지)**

  - 서버가 전달한 데이터가 **브라우저 내부에 저장**되어,
  클라이언트가 이를 필요할 때 직접 사용하도록 설계된 저장소.
  - HTTP 요청 시 자동으로 전송되지 않으며, **단순한 데이터 저장 목적**이다.


 > **Local Storage (로컬 스토리지)**

* 데이터를 **영구적으로 저장**한다.
* 브라우저를 닫아도 유지됨.
* 예: 사용자 테마, 최근 검색어


 > **Session Storage (세션 스토리지)**


  * **브라우저 탭(세션) 동안만 유지되는 일시적 저장소.**
  * 탭을 닫으면 삭제된다.
  * 예: 현재 입력 중인 폼 데이터

---

##  한 줄 비교 요약

| 구분                  | 저장 위치          | 유지 기간       | 자동 전송 여부        | 주요 용도             |
| ------------------- | -------------- | ----------- | --------------- | ----------------- |
| **Cache**           | 로컬 PC (브라우저)   | 브라우저 설정에 따름 | ❌               | 리소스 재활용 (성능 최적화)  |
| **Cookie**          | 브라우저 (파일)      | 설정된 만료일     | ✅               | 로그인 세션 식별, 상태 유지  |
| **Session**         | 서버 (메모리/Redis) | 세션 종료 시     | 🔄 쿠키의 세션ID로 매핑 | 사용자 상태 정보 관리      |
| **Local Storage**   | 브라우저           | 영구적         | ❌               | 사용자 설정, 비로그인 데이터  |
| **Session Storage** | 브라우저           | 탭 닫을 때까지    | ❌               | 임시 데이터 (입력값 유지 등) |
